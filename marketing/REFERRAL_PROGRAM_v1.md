# 친구 추천 프로그램 기획 v1 (재설계 2026-05-21)

> 베트남 한인 비즈니스 특수성 반영 — 잡지는 이미 모든 구독자에게 *무료 우송*이고
> 광고는 교민에게 *불편 아닌 정보*. 따라서 보상은 **광고주 협력 매장 쿠폰**만으로 한정.
> 씬짜오 비용 0, 광고주 부담, 사용자 실질 가치 — 3자 윈윈 구조.
>
> 작성: 2026-05-21 (사용자 피드백 반영 v1)
> 마케팅 활성화 방안 §3 액션 9 — 31~60일 단계
>
> 이 문서는 *기획 단계*. 구현은 측정 데이터 정착 후 (~2~4주).

---

## 1. 한 줄 컨셉

> **"친구 N명을 씬짜오 앱에 초대하면, 동네 한국 식당·미용실 쿠폰을 드려요."**

- **씬짜오 비용**: 거의 0 (광고주 부담)
- **광고주 가치**: 매장 신규 손님 유입 + 잡지 광고 효과 측정 자료
- **사용자 가치**: 실제 사용 가능한 쿠폰 (음료 무료·식사 할인·미용 쿠폰 등)

## 2. 왜 *이런* 보상인가 (제외한 후보들)

| 후보 | 채택 안 한 이유 |
|---|---|
| 잡지 1년 무료 구독 | ❌ 이미 모든 구독자에게 무료 우송 중 — 보상 아닌 기본 서비스 |
| 앱 광고 제거 1개월 | ❌ 교민에게 광고는 *불편*이 아닌 *정보* (한국 식당·미용실·부동산). 제거는 사용자 가치 감소 |
| 현금/포인트 | ❌ 한계비용 큼. 어뷰징 위험 |
| 매장 쿠폰 (광고주 협력) | ✅ 씬짜오 비용 0. 광고주 매장 손님 ↑. 사용자 실질 가치. 3자 윈윈 |

## 3. 보상 단계 (3 트랙)

### 🥉 트랙 1: 친구 1명 가입 → *디지털 가치*
- 앱 내 "추천 뱃지" 1개
- 환영 화면에 "이 사용자는 친구를 데려온 회원이에요" 작은 표시
- **한계비용**: 0
- **목적**: 가벼운 출발선 — 사람들이 추천 행동을 *시작*하게

### 🥈 트랙 2: 친구 3명 가입 → *동네 매장 쿠폰 1장*
- 예시: "한국 식당 ○○ 음료 1잔 무료" 또는 "미용실 ○○ 10% 할인"
- 광고주 매장에서 직접 사용. 코드 + 사용자 닉네임을 앱에서 보여주고 적용
- **광고주 비용**: 쿠폰 1건당 원가 약 5천~2만 동 (음료/할인 등)
- **목적**: 메인 보상 — 추천 행동의 핵심 동기

### 🥇 트랙 3: 친구 10명 가입 → *프리미엄 쿠폰 패키지*
- 예시: 한국 식당 식사 50% 할인 + 미용실 1회 무료 등 3~5장 묶음
- 광고주 협력 5~10곳 중 사용자가 *고를 수 있는* 선택권
- **목적**: 추천 챔피언 인정 + 광고주 5~10곳 전부 손님 유입

## 4. 광고 바우처 *연결* 흐름 (사용자 의문 해결)

> **3자 (씬짜오 / 광고주 / 사용자) 어떻게 연결되는가** — v1 은 *수동 운영*으로 시작.

### Step 1. 광고주 영업 (씬짜오 → 광고주)

- 잡지 광고주 풀에서 *매장 운영자* 선별 (식당·미용실·마트·여행사 등)
- 협력 제안: **"광고주 매장에 새 손님 유입을 무료로 만들어드립니다. 추천 보상 쿠폰 발행에 참여하시면…"**
  - 광고주 부담: 쿠폰 1건당 원가만 (음료 무료 등)
  - 씬짜오 제공: 월간 쿠폰 사용 수 리포트 + 매장 노출 (앱 내 매장 정보)
- 협력 매장 *5~10곳* 모집 → v1 시작

### Step 2. 쿠폰 등록 (광고주 → 씬짜오 시스템)

- 광고주가 *쿠폰 내용* 결정: "음료 1잔 무료" / "10% 할인" / "1+1" 등
- 씬짜오 어드민 페이지에서 *쿠폰 등록*: 광고주명·쿠폰 내용·매장 주소·유효기간·월 발행 한도
- Firestore `coupons` 컬렉션에 저장

### Step 3. 추천 발생 (사용자 → 친구)

- 사용자가 앱 → 메뉴 → "친구 초대" 진입
- 자신의 추천 코드(예: `XCV-A4B7`) + 공유 링크 표시
- 카톡/메시지로 친구에게 전송 → 친구가 링크로 앱 설치 + 가입
- 가입 시 추천 코드 자동 인식 → Firestore에 `referrals` 레코드 생성

### Step 4. Verified 검증 (씬짜오 시스템 자동)

- 추천받은 친구가 *7일 동안 앱 1회 이상 사용* → `status: verified`
- 즉시 보상 X (어뷰징 방지)
- Verified 누적 3건 시 → *쿠폰 1장 자동 발급* (트랙 2)

### Step 5. 쿠폰 사용 (사용자 → 광고주 매장)

- 사용자가 앱 → 메뉴 → "내 쿠폰" 진입
- 쿠폰 화면: **광고주 매장 정보 + 쿠폰 내용 + 사용자 닉네임 + 8자리 코드(예: `XCV-A4B7-CPN1`)**
- 매장 방문 → 직원에게 *앱 화면 보여줌*
- 매장 직원: **카운터 메모장에 닉네임 + 쿠폰 코드 + 날짜 적기** + 할인/혜택 적용

### Step 6. 월말 정산 (광고주 → 씬짜오)

- 매장이 *월말에 카운터 메모장 사진*을 씬짜오에 전송 (카톡 또는 이메일)
- 씬짜오 어드민이 *사용 수 카운트* → 광고주에게 효과 리포트 발송
- 광고주: 손님 유입 데이터 확보 → 잡지 광고 가치 증명

> **부정 방지**: 매장 직원이 *앱 화면 + 닉네임*까지 확인하면 사용자가 코드만 복사해서 친구에게 주는 어뷰징은 어려움. 완벽 방지 X, 단 광고주 단가가 낮아 손해 미미.

## 5. 시스템 설계 (구현 시점 참고)

### Firestore 스키마

```js
// coupons 컬렉션 — 광고주가 등록한 쿠폰 정보
{
  id: "auto",
  advertiserId: "광고주 식별",
  advertiserName: "한국식당 ○○",
  storeAddress: "호치민 떤덕군 ...",
  couponName: "음료 1잔 무료",
  couponDesc: "한국 식당 ○○에서 음료(콜라/사이다) 1잔 무료 (식사 주문 시)",
  validityDays: 60,           // 발급일로부터 60일 유효
  monthlyLimit: 50,           // 월간 발행 한도
  monthlyIssued: 0,           // 현재 월 발행 수 (자동 reset)
  tier: 2,                    // 1·2·3 트랙 매핑
  isActive: true,
}

// referrals 컬렉션
{
  id: "auto",
  referrerUid: "추천한 사람",
  refereeUid: "가입한 친구",
  status: "pending" | "verified",
  createdAt, verifiedAt,
}

// userCoupons 컬렉션 — 사용자가 보유한 쿠폰
{
  id: "auto",
  userUid: "사용자",
  couponId: "쿠폰 원본 ID",
  code: "XCV-A4B7-CPN1",      // 8자리 + 사용자별 unique
  issuedAt: serverTimestamp(),
  expiresAt: timestamp,
  usedAt: null,                // 매장이 사용 처리 시 갱신
  usedAtStore: null,
}

// users 확장
{
  referralCode: "XCV-A4B7",     // 가입 시 자동 생성
  referredBy: "XCV-XXXX" | null,
  referralCount: 0,             // verified 추천 수 누적
}
```

### 앱 화면 (4개)

1. **친구 초대**: 추천 코드·공유 링크·진행 상황 ("2/3명 가입. 1명만 더 데려오면 쿠폰!")
2. **내 쿠폰**: 보유 쿠폰 리스트 (광고주 + 내용 + 코드 + 유효기간)
3. **쿠폰 상세**: 큰 화면에 코드 + 닉네임 + 광고주 매장 정보 (지도 포함)
4. **환영 화면 확장** (이미 만든 A v2): 추천받아 가입한 경우 "[추천인 닉네임]님이 초대했어요" 표시

### 어드민 페이지 (3개)

1. **쿠폰 등록**: 광고주가 직접 또는 씬짜오 운영진이 대신 등록
2. **매장 사용 보고**: 매장 직원이 *월말 보고서 업로드* (메모장 사진 + 사용 수)
3. **쿠폰 효과 리포트**: 광고주별 발행 수·사용 수·사용 매장 (월간)

## 6. 단계별 구현 로드맵

### Phase 1 — 광고주 협력 모집 (1~2주, **씬짜오 운영진**)
- 잡지 광고주 풀에서 5~10곳 협력 매장 모집
- 협력 조건: 쿠폰 1건당 원가 부담 (할인/무료) + 월말 사용 보고
- 첫 매장은 *기존 정기 광고주* 중 의리 좋은 곳 1~2곳 — 시범 운영
- 산출물: 협력 매장 리스트 + 쿠폰 내용 합의서

### Phase 2 — 시스템 구현 (1~2주, 개발)
- Firestore 스키마 + Cloud Function (referral verified 처리, 쿠폰 자동 발급)
- 앱 화면 4개 신규 + 어드민 페이지 3개
- Universal Link (deferred deep link) — *네이티브 모듈 추가 필요* → EAS Build
- 추천 코드 생성·검증 로직

### Phase 3 — 운영 시작 (1주, 시범)
- 첫 매장 1~2곳으로 시작
- 사용자 100명 정도 시범 — 어뷰징·UX 검증
- 보고 양식 운영 — 매장 직원 트레이닝

### Phase 4 — 확장 (지속)
- 협력 매장 30곳 목표
- 광고주 자가 등록 어드민 페이지 (Phase 2 의 어드민 자기 사용)
- 사용 보고 자동화 (사진 → OCR? → 매장이 *간단 어드민*에서 직접 처리)

## 8. 위험·한계

- **광고주 참여 모집 실패** — 잡지 광고주가 *추가 부담*을 거부할 수 있음
  → 첫 1~2곳은 *의리 좋은 광고주* + 시범 결과 데이터 보여주고 확장
- **매장 직원이 보고 안 함** — 월말 사진 보고 누락
  → 광고주에게 *효과 리포트*가 인센티브. 보고 안 하면 리포트 X
- **사용자가 쿠폰만 노리고 어뷰징** — 가짜 가입 친구
  → 7일 retention 검증 + 한 사용자 월 3건 verified 한도 + 디바이스 fingerprint
- **베트남 한인 시장 포화** — 호치민 8만 + 하노이 3만 = 11만. 추천 가능 인구 한정. 1~2년 내 포화 예상
